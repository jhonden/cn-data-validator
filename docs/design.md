## 1. 项目概述

### 1.1 项目背景

网络架构评估工具需要用户从现网采集网络数据，导入系统进行解析还原。但在采集过程中，容易出现以下问题：

- **数据包漏采**：采集的数据包不完整
- **文件缺失**：数据包内部的必需文件缺失
- **格式非法**：采集的文件格式不符合要求

这些问题会影响后续的数据还原和分析工作，因此需要一个本地校验工具，让用户在采集完成后立即检查数据质量。

### 1.2 项目目标

开发一个本地校验工具，用于：
- ✅ 校验数据包文件格式是否合法
- ✅ 识别数据包类型（NIC/LCM/NFVI 等）
- ✅ 检查数据包内部文件的完整性
- ✅ 检查数据包之间的依赖关系
- ✅ 生成校验报告，便于提交给技术支持

### 1.3 使用场景

1. **现场采集后校验**：一线人员采集完数据后，立即运行校验工具
2. **批量导入前检查**：将数据导入系统前，先运行校验工具
3. **技术支持诊断**：遇到问题时，先运行校验工具定位问题

## 3. 已实现功能

### 3.1 文件格式校验

**功能描述**：校验文件格式是否为支持的类型

**支持的格式**：
- ✅ ZIP (`.zip`)
- ✅ TAR (`.tar`)
- ✅ TAR.GZ (`.tar.gz`)
- ✅ Excel (`.xlsx`)

**实现细节**：
- 递归扫描目录及其所有子目录
- 检查每个文件的扩展名
- 区分合法文件和非法文件
- 计算文件大小和修改时间

**数据结构**：
```python
{
    'path': '/full/path/to/file.tar.gz',
    'relative_path': 'subdir/file.tar.gz',
    'name': 'file.tar.gz',
    'size': 12345,
    'modified': datetime(2025, 3, 5, 10, 30, 45),
    'package_type': 'NIC包' 或 '未知',
    'package_details': { ... }
}
```

**校验结果**：
- ✅ 合法：文件格式符合要求
- ❌ 非法：文件格式不符合要求，提示"数据包格式非法"

### 3.2 数据包类型识别

**功能描述**：识别不同类型的数据包，目前支持 NIC 包识别

#### NIC 包识别规则

**识别条件**（必须全部满足）：
1. 文件格式：`.tar.gz`
2. 解压后第一层级包含：
   - 时间格式命名的文件夹：`YYYYMMDDHHMMSS`（14位数字）
   - 对应的报告文件：`{时间}_report.tar.gz`
3. 文件数量：第一层级只有这 2 个项目

**示例结构**：
```
nic_package.tar.gz
├── 20250203105511/              # 时间格式文件夹
│   └── ...                        # 时间文件夹内部内容
└── 20250203105511_report.tar.gz  # 报告压缩文件
```

**识别细节**：
- 使用正则表达式 `^\d{14}$` 验证时间格式
- 使用 `datetime.strptime()` 验证时间的有效性
- 严格匹配报告文件名 `{时间}_report.tar.gz`

**识别结果**：
```python
{
    'package_type': 'NIC包',
    'package_details': {
        'time': '20250203105511',
        'report_file': '20250203105511_report.tar.gz',
        'structure': '时间文件夹 + 报告文件'
    }
}
```

**未识别结果**：
```python
{
    'package_type': '未知',
    'package_details': {
        'note': '无法识别的数据包类型' 或 '不支持的文件格式'
    }
}
```

### 3.4 NIC 包深度校验

**功能描述**：对识别为 NIC 包的数据包进行深度校验，检查网元元数据和关键文件完整性。

#### 3.4.1 neinfo.txt 文件解析

**文件格式**：
```
PPR_CSCF01   CSCF_NE=1094   56   10.140.2.1
PPR_ATS01   ATS_NE=1044   59   10.140.2.10
```

**字段说明**：
| 字段 | 说明 | 示例 |
|------|------|------|
| 网元实例名称 | NE 实例标识符 | PPR_CSCF01 |
| 网元类型_实例ID | 格式为 `类型=ID` | CSCF_NE=1094 |
| 组 ID | 网元所属组 | 56 |
| IP 地址 | 网元 IP 地址 | 10.140.2.1 |

**支持的网元类型**：
```
UNC, UDG, vCG, vUGW, vUSN, ATS, CSCF, CloudSE2980, SE2900,
CCF, SPSV3, USC, HSS9860, UDM, UPCC, UPCF, ENS, USCDB, CSP, CGPOMU
```

---

#### 3.4.2 校验规则

##### 校验规则1：网元类型支持检查

**规则描述**：当 NIC 包中所有网元类型都不在 CNAE 支持范围内时，校验失败。

**判断逻辑**：
- 如果 NIC 包中至少有一个网元类型在支持列表中 → **校验通过**（可能带有警告）
- 如果 NIC 包中所有网元类型都不在支持列表中 → **校验失败**

**示例场景**：
```
场景1：包中包含支持的网元类型
- NE1: CSCF_NE=1094 (支持) ✅
- NE2: ATS_NE=1044 (支持) ✅
结果：Valid（可能有数据完整性警告）

场景2：包中包含部分支持的网元类型
- NE1: CSCF_NE=1094 (支持) ✅
- NE2: UNKNOWN_NE=9999 (不支持) ⚠️
结果：Valid，提示有 1 个不支持的网元类型

场景3：包中所有网元类型都不支持
- NE1: UNSUPPORTED_NE1=9999 (不支持) ❌
- NE2: UNSUPPORTED_NE2=9998 (不支持) ❌
结果：Invalid，提示：No CNAE-supported network elements found in this NIC package, no need to collect
```

**显示逻辑**：
| 条件 | 状态 | 颜色 | 提示信息 |
|------|------|------|----------|
| 有支持的网元类型 | Valid | 绿色 #4CAF50 | - |
| 所有网元类型都不支持 | Invalid | 红色 #F44336 | No CNAE-supported network elements found in this NIC package, no need to collect |
| 有不支持的网元（部分支持） | Warning | 橙色 #FF9800 | Unsupported NE types: {types} |

---

##### 校验规则2：neinfo.txt 文件检查

**规则描述**：NIC 包必须包含 neinfo.txt 文件，否则格式非法。

**判断逻辑**：
- neinfo.txt 存在 → 继后续校验
- neinfo.txt 不存在 → **校验失败**

**显示逻辑**：
| 条件 | 状态 | 颜色 | 提示信息 |
|------|------|------|----------|
| neinfo.txt 不存在 | Invalid | 红色 #F44336 | Invalid NIC package format: Missing required file neinfo.txt |

---

##### 校验规则3：采集时间范围检查

**规则描述**：校验 NIC 包的采集时间范围是否大于 24 小时。

**判断逻辑**：
- 采集时间范围 ≥ 24 小时 → **校验通过**（可能带有其他警告）
- 采集时间范围 < 24 小时 → **校验失败**

**数据来源**：
- 从 `xxx_report.tar.gz` 中的 `taskparam/TaskExtValue.xml` 中解析
- 查找 `CollectRange` 标签，格式：开始时间|结束时间
- 示例：`2025-03-20 21:15:18|2025-03-27 21:15:18`
- 计算时间差（小时）

**示例场景**：
```
场景1：采集时间范围 ≥ 24 小时
- CollectRange: 2025-03-20 21:15:18|2025-03-27 21:15:18
- 时间差：168 小时（7 天）
结果：Valid（可能有其他警告）

场景2：采集时间范围 < 24 小时
- CollectRange: 2025-03-20 21:15:18|2025-03-21 09:45:18
- 时间差：12.5 小时
结果：Invalid，提示：NIC package collection time range is too short, cannot support
       network assessment requirements. System requires at least 24h, please re-collect.
       (Actual: 12.50h)
```

**显示逻辑**：
| 条件 | 状态 | 颜色 | 提示信息 |
|------|------|------|----------|
| 采集时间范围 < 24h | Invalid | 红色 #F44336 | NIC package collection time range is too short, cannot support network assessment requirements. System requires at least 24h, please re-collect. (Actual: 12.50h) |
| 采集时间范围 ≥ 24h | Valid | 绿色 #4CAF50 | - |

---

##### 校验规则4：匿名化方式采集检查

**规则描述**：校验 NIC 包是否使用匿名化方式采集。

**判断逻辑**：
- 非匿名化（AnonymousAuthMode = false）→ **校验通过**
- 匿名化（AnonymousAuthMode = true）→ **校验失败**

**数据来源**：
- 从 `xxx_report.tar.gz` 中的 `taskparam/TaskExtValue.xml` 中解析
- 查找 `AnonymousAuthMode` 标签
- `true` 代表匿名化，`false` 代表非匿名化

**示例场景**：
```
场景1：非匿名化采集
- AnonymousAuthMode: false
结果：Valid（可能有其他警告）

场景2：匿名化采集
- AnonymousAuthMode: true
结果：Invalid，提示：The NIC package was collected in anonymous mode, cannot meet
       network assessment requirements. Please re-collect in non-anonymous mode.
```

**显示逻辑**：
| 条件 | 状态 | 颜色 | 提示信息 |
|------|------|------|----------|
| 匿名化采集（true） | Invalid | 红色 #F44336 | The NIC package was collected in anonymous mode, cannot meet network assessment requirements. Please re-collect in non-anonymous mode. |
| 非匿名化采集（false） | Valid | 绿色 #4CAF50 | - |

## 4. 数据流程

### 4.1 扫描流程

```
开始
  ↓
用户选择目录
  ↓
[FileScanner.scan_directory()]
  ↓
遍历目录树（os.walk）
  ↓
对每个文件：
  ├─ 检查文件格式
  │   ├─ 合法 → 添加到 valid_files
  │   └─ 非法 → 添加到 illegal_files
  │
  └─ [PackageIdentifier.identify()]
      ├─ 解压文件（如需要）
      ├─ 分析内部结构
      ├─ 识别包类型
      └─ 返回 (package_type, package_details)
  ↓
扫描完成，返回统计数据
  ↓
GUI/CLI 显示结果
  ↓
用户可选导出结果
  ↓
结束
```

### 4.2 包识别流程

```
[PackageIdentifier.identify()]
  ↓
检查文件扩展名
  ├─ .tar.gz → _identify_tar_gz()
  ├─ .zip → _identify_zip()
  ├─ .tar → _identify_tar()
  └─ .xlsx → (未知，Excel 文件)
  ↓
[针对 .tar.gz 文件]
  ├─ 创建临时目录
  ├─ 解压文件（tarfile.extractall）
  ├─ 获取第一层级文件列表
  ├─ 验证文件数量 == 2
  ├─ 验证包含 1 个文件夹 + 1 个文件
  ├─ 验证文件夹名符合时间格式
  ├─ 验证时间有效性
  ├─ 验证报告文件名匹配 {时间}_report.tar.gz
  ├─ 全部通过 → 识别为 NIC包
  └─ 任何失败 → 未识别
  ↓
清理临时目录
  ↓
返回 (package_type, package_details)
```

## 9. 已知问题和限制

### 9.1 已知问题

1. **macOS Tkinter 兼容性**
   - 问题描述：macOS 15 (1507) or later required, have instead 15 (1506)
   - 影响：CustomTkinter GUI 无法在特定 macOS 版本运行
   - 解决方案：使用 PyQt6 或 PyInstaller 打包

2. **PyInstaller 打包体积**
   - 问题描述：PyQt6 打包后文件较大（50-100MB）
   - 影响：用户下载和分发时文件较大
   - 解决方案：无（Qt 框架特性）

### 9.2 当前限制

1. **包识别规则硬编码**
   - 限制：目前 NIC 包识别规则硬编码在代码中
   - 影响：添加新包类型需要修改代码
   - 计划：支持配置文件（P2 优先级）

2. **单次扫描无增量**
   - 限制：每次都需要重新扫描整个目录
   - 影响：大型目录扫描较慢
   - 计划：支持缓存和增量扫描（P2 优先级）
