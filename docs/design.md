## 1. 项目概述

### 1.1 项目背景

网络架构评估工具需要用户从现网采集网络数据，导入系统进行解析还原。但在采集过程中，容易出现以下问题：

- **数据包漏采**：采集的数据包不完整
- **文件缺失**：数据包内部的必需文件缺失
- **格式非法**：采集的文件格式不符合要求

这些问题会影响后续的数据还原和分析工作，因此需要一个本地校验工具，让用户在采集完成后立即检查数据质量。

### 1.2 项目目标

开发一个本地校验工具，用于：
- ✅ 校验数据包文件格式是否合法
- ✅ 识别数据包类型（NIC/LCM/NFVI 等）
- ✅ 检查数据包内部文件的完整性
- ✅ 检查数据包之间的依赖关系
- ✅ 生成校验报告，便于提交给技术支持

### 1.3 使用场景

1. **现场采集后校验**：一线人员采集完数据后，立即运行校验工具
2. **批量导入前检查**：将数据导入系统前，先运行校验工具
3. **技术支持诊断**：遇到问题时，先运行校验工具定位问题

## 3. 已实现功能

### 3.1 文件格式校验

**功能描述**：校验文件格式是否为支持的类型

**支持的格式**：
- ✅ ZIP (`.zip`)
- ✅ TAR (`.tar`)
- ✅ TAR.GZ (`.tar.gz`)
- ✅ Excel (`.xlsx`)

**实现细节**：
- 递归扫描目录及其所有子目录
- 检查每个文件的扩展名
- 区分合法文件和非法文件
- 计算文件大小和修改时间

**数据结构**：
```python
{
    'path': '/full/path/to/file.tar.gz',
    'relative_path': 'subdir/file.tar.gz',
    'name': 'file.tar.gz',
    'size': 12345,
    'modified': datetime(2025, 3, 5, 10, 30, 45),
    'package_type': 'NIC包' 或 '未知',
    'package_details': { ... }
}
```

**校验结果**：
- ✅ 合法：文件格式符合要求
- ❌ 非法：文件格式不符合要求，提示"数据包格式非法"

---

### 3.2 数据包类型识别

**功能描述**：识别不同类型的数据包，目前支持 NIC 包识别

#### NIC 包识别规则

**识别条件**（必须全部满足）：
1. 文件格式：`.tar.gz`
2. 解压后第一层级包含：
   - 时间格式命名的文件夹：`YYYYMMDDHHMMSS`（14位数字）
   - 对应的报告文件：`{时间}_report.tar.gz`
3. 文件数量：第一层级只有这 2 个项目

**示例结构**：
```
nic_package.tar.gz
├── 20250203105511/              # 时间格式文件夹
│   └── ...                        # 时间文件夹内部内容
└── 20250203105511_report.tar.gz  # 报告压缩文件
```

**识别细节**：
- 使用正则表达式 `^\d{14}$` 验证时间格式
- 使用 `datetime.strptime()` 验证时间的有效性
- 严格匹配报告文件名 `{时间}_report.tar.gz`

**识别结果**：
```python
{
    'package_type': 'NIC包',
    'package_details': {
        'time': '20250203105511',
        'report_file': '20250203105511_report.tar.gz',
        'structure': '时间文件夹 + 报告文件'
    }
}
```

**未识别结果**：
```python
{
    'package_type': '未知',
    'package_details': {
        'note': '无法识别的数据包类型' 或 '不支持的文件格式'
    }
}
```

## 4. 数据流程

### 4.1 扫描流程

```
开始
  ↓
用户选择目录
  ↓
[FileScanner.scan_directory()]
  ↓
遍历目录树（os.walk）
  ↓
对每个文件：
  ├─ 检查文件格式
  │   ├─ 合法 → 添加到 valid_files
  │   └─ 非法 → 添加到 illegal_files
  │
  └─ [PackageIdentifier.identify()]
      ├─ 解压文件（如需要）
      ├─ 分析内部结构
      ├─ 识别包类型
      └─ 返回 (package_type, package_details)
  ↓
扫描完成，返回统计数据
  ↓
GUI/CLI 显示结果
  ↓
用户可选导出结果
  ↓
结束
```

### 4.2 包识别流程

```
[PackageIdentifier.identify()]
  ↓
检查文件扩展名
  ├─ .tar.gz → _identify_tar_gz()
  ├─ .zip → _identify_zip()
  ├─ .tar → _identify_tar()
  └─ .xlsx → (未知，Excel 文件)
  ↓
[针对 .tar.gz 文件]
  ├─ 创建临时目录
  ├─ 解压文件（tarfile.extractall）
  ├─ 获取第一层级文件列表
  ├─ 验证文件数量 == 2
  ├─ 验证包含 1 个文件夹 + 1 个文件
  ├─ 验证文件夹名符合时间格式
  ├─ 验证时间有效性
  ├─ 验证报告文件名匹配 {时间}_report.tar.gz
  ├─ 全部通过 → 识别为 NIC包
  └─ 任何失败 → 未识别
  ↓
清理临时目录
  ↓
返回 (package_type, package_details)
```

## 9. 已知问题和限制

### 9.1 已知问题

1. **macOS Tkinter 兼容性**
   - 问题描述：macOS 15 (1507) or later required, have instead 15 (1506)
   - 影响：CustomTkinter GUI 无法在特定 macOS 版本运行
   - 解决方案：使用 PyQt6 或 PyInstaller 打包

2. **PyInstaller 打包体积**
   - 问题描述：PyQt6 打包后文件较大（50-100MB）
   - 影响：用户下载和分发时文件较大
   - 解决方案：无（Qt 框架特性）

### 9.2 当前限制

1. **包识别规则硬编码**
   - 限制：目前 NIC 包识别规则硬编码在代码中
   - 影响：添加新包类型需要修改代码
   - 计划：支持配置文件（P2 优先级）

2. **单次扫描无增量**
   - 限制：每次都需要重新扫描整个目录
   - 影响：大型目录扫描较慢
   - 计划：支持缓存和增量扫描（P2 优先级）
